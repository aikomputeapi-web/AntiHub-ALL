# AntiHub 部署手册

## 前置要求

- 已安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（或 Docker Engine + Docker Compose v2）
- 内存建议 ≥ 2GB

---

## 第一步：配置环境变量

```bash
cp .env.example .env
```

打开 `.env`，**必须修改**以下几项：

| 变量 | 说明 | 示例 |
|------|------|------|
| `JWT_SECRET_KEY` | JWT 签名密钥，任意强随机字符串 | `openssl rand -base64 32` 生成 |
| `PLUGIN_API_ENCRYPTION_KEY` | Fernet 加密密钥 | 见下方生成命令 |
| `ADMIN_USERNAME` | 首次启动自动创建的管理员账号 | `admin` |
| `ADMIN_PASSWORD` | 管理员密码，请改成强密码 | `MyStr0ngPass!` |

**生成 PLUGIN_API_ENCRYPTION_KEY（二选一）：**
```bash
# 方法1：Python（需安装）
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# 方法2：构建完成后用容器生成
docker compose -f docker-compose.yml -f docker-compose.local.yml run --rm backend python generate_encryption_key.py
```

## 方案一：对方 Docker 里没有数据库和 Redis（全新部署，推荐）

> 会一并启动 postgres、redis、backend、web，一键搞定。

### 启动

```bash
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build
```

> 首次构建需要下载依赖，约需 3~10 分钟（视网络速度）。

### 如果 5432 端口被占用

在 `.env` 中修改：
```
POSTGRES_PORT=5433
```
然后重新执行上面的启动命令。

## 方案二：对方 Docker 里已有 Postgres 和 Redis

> 只起 web 和 backend，复用现有数据库。

### 1. 确认容器网络

先查看已有 postgres/redis 容器在哪个网络：
```bash
docker inspect <postgres容器名> --format '{{json .NetworkSettings.Networks}}' | python3 -m json.tool | grep -o '"[^"]*"' | head -5
```

**两种处理方式（选一）：**

**A. 把已有容器加入 antihub 网络（推荐）：**
```bash
docker network create antihub-core_default 2>/dev/null || true
docker network connect antihub-core_default <postgres容器名>
docker network connect antihub-core_default <redis容器名>
```
然后修改 `docker-compose.core.yml` 底部的网络名：
```yaml
networks:
  default:
    external: true
    name: antihub-core_default   # 改成你创建的网络名
```

**B. 直接用 IP/容器名写入 .env（最简单）：**
```
DATABASE_URL=postgresql+asyncpg://<用户名>:<密码>@<postgres容器名>:5432/<数据库名>
REDIS_URL=redis://<redis容器名>:6379/0
```

### 2. 启动

```bash
docker compose -f docker-compose.core.yml up -d --build
```

## 验证

```bash
# 查看容器状态（backend 状态应为 healthy）
docker ps

# 检查后端健康接口（返回 {"status":"healthy"...} 即正常）
curl http://localhost:8000/api/health

# 打开前端
open http://localhost:3000       # macOS
# 或浏览器访问 http://localhost:3000
```

## 访问地址

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:3000 |
| 后端 API 文档 | http://localhost:8000/api/docs |

默认管理员账号：`.env` 中 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 所配置的值。

## 停止 / 重启

```bash
# 方案一（完整）
docker compose -f docker-compose.yml -f docker-compose.local.yml down     # 停止（保留数据）
docker compose -f docker-compose.yml -f docker-compose.local.yml down -v  # 停止并清空数据库

# 方案二（core）
docker compose -f docker-compose.core.yml down
```

## 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| `backend` 一直 Restarting | 密码/连接配置错误 | `docker logs antihub-backend` 看错误信息 |
| `5432 bind: address already in use` | 宿主机 Postgres 占用端口 | `.env` 改 `POSTGRES_PORT=5433` |
| `JWT_SECRET_KEY` 未配置报错 | `.env` 未设置 | 见第一步必填项 |
| 前端登录后报 502 | backend 未健康 | `docker ps` 检查 backend 状态 |
